What is Server Guard?

Server Guard is a bot made to protect Guilded servers from trolls, ban evaders, and raids/bots. All users who join the server are asked to verify themselves.

How does Server Guard work?

Server Guard works by scanning a user’s Guilded connections for anything suspicious. Server Guard will also perform a check to see if a user’s IP matches a hashed IP of any previously banned user to help protect against ban evasion! We use metrics such as Steam account level or Roblox account age for our checks.

What is the Default Role?

The Default Role in Server Guard’s context means a role that is given to all members in your server when they join. This role is taken away once the user has verified. The reverse can also be done, so that it acts like a verification role instead.

What are the Admin and Mod roles?

These are roles you can add to the bot which give access to permissions within the bot. They are both near identical, with the only difference being that Admins can change the bot’s settings, while Mods are unable to.

What are the verification and log channels?

The log channel is where Server Guard will log the status of new users going through verification. The bot needs a place for new users to verify/be notified of the need to verification, which the verification channel is for. The verification channel should be the only channel an unverified user can interact with or see and is required.
